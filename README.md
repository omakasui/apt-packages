# omakasui/apt-packages

Custom APT repository for Ubuntu 24.04 (noble) and Debian 13 (trixie).
Packages are built by [omakasui/build-packages](https://github.com/omakasui/build-apt-packages)
and published here automatically via CI.

## Installation

```bash
# Import GPG signing key
curl -fsSL https://packages.omakasui.org/omakasui.gpg.key \
  | gpg --dearmor \
  | sudo tee /usr/share/keyrings/omakasui-apt.gpg > /dev/null

# Add APT source
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/omakasui-apt.gpg] \
  https://packages.omakasui.org $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}") main" \
  | sudo tee /etc/apt/sources.list.d/omakasui.list

sudo apt update
sudo apt install <package>
```

## Repository structure

```
omakasui.gpg.key              ← public GPG key (auto-regenerated by CI)
dists/<suite>/
  Release / Release.gpg / InRelease
  main/binary-{amd64,arm64}/Packages(.gz/.xz)
pool/<suite>/
  all/                        ← arch-independent .deb files
  amd64/ arm64/               ← arch-specific .deb files
```

`.deb` files are committed to Git so GitHub Pages can serve them directly.
`*.deb` is in `.gitignore`; CI uses `git add -f` to force-track them.

## Maintenance

### Adding a new distro

1. Add it to `build-matrix.yml` in `omakasui/build-apt-packages`.
2. In this repo's workflow ([.github/workflows/update.yml](.github/workflows/update.yml)), update:
   - `env.ALL_SUITES` — append the new codename
   - `DISTRO_MAP` in the download step — map codename to the build-apt-packages distro label (e.g. `[oracular]="ubuntu2410"`)

### Manual operations

GitHub → Actions → **Update APT repo** → Run workflow.

| Action | When to use |
|--------|-------------|
| `package-updated` | Re-download and re-index a specific version. Requires package, version, suites. |
| `remove-package` | Remove a package from the pool and regenerate indexes. Suites optional (defaults to all). |
| `rebuild-index` | Regenerate APT metadata from current pool contents, no pool changes. |
